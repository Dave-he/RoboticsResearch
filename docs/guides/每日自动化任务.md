## 当前在用的方案:systemd user timer (2026-07-01 起)

参考 LNN 项目的 `lnn-daily-research.{timer,service}` 改写。

### 组件

- **入口脚本**:`scripts/run_daily_robotics_task.sh`(本仓库)
  - 调 `python3 scripts/daily_robotics_research.py ...`
  - 若有变更则 `git add docs papers analysis` + `git commit` + `git push`
- **systemd service**:`~/.config/systemd/user/robotics-daily-research.service`
  - `Type=oneshot`,`WorkingDirectory=/mnt/ssd/codespace/research/RoboticsResearch`
  - 通过 environment 注入 `MAX_ARXIV=80`、`COMMIT_AND_PUSH=1` 等参数
- **systemd timer**:`~/.config/systemd/user/robotics-daily-research.timer`
  - `OnCalendar=*-*-* 07:00:00`(故意比 LNN 的 06:30 错开 30 分钟,避免同时打 arXiv)
  - `RandomizedDelaySec=10m`
  - `Persistent=true`(机器关机后下次开机自动补跑错过的任务)

### 安装命令

```bash
# 1. 让入口脚本可执行
chmod +x /mnt/ssd/codespace/research/RoboticsResearch/scripts/run_daily_robotics_task.sh

# 2. systemd 重载 + 启用 timer(已在本机执行)
systemctl --user daemon-reload
systemctl --user enable --now robotics-daily-research.timer

# 3. 查看下次触发时间
systemctl --user list-timers | grep robotics

# 4. 立即试跑一次(可选)
systemctl --user start robotics-daily-research.service
journalctl --user -u robotics-daily-research.service -n 50
```

### 行为约定

- 每天 07:00 CST(±10 min 抖动)自动跑,失败无回滚,下次会覆盖
- 提交信息为 `chore(daily): update RoboticsResearch digest <YYYY-MM-DD>`
- 只覆盖 fetch 部分(`papers/daily/` + `docs/daily/` + `analysis/repo_watchlist/` + 全局索引),**深度研读报告**仍由人工 / interactive session 触发(通过 `/goal` 进入日研究流程时一并产出)
