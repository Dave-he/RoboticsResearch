# <论文标题>

## 元数据

- **arXiv ID**: <ID>
- **作者**: <第一作者 等 et al.>
- **机构**: <Stanford / Google DeepMind / ...>
- **发表时间**: YYYY-MM
- **会议 / 期刊**: <如 ICRA 2025 / CoRL 2024>
- **项目页**: <URL>
- **代码**: <GitHub URL 如有>
- **PDF**: [link](https://arxiv.org/pdf/<ID>)

## 核心问题

> 论文要解决的具体痛点(2-4 句)。不要写"研究 XX 任务"这种泛泛的,要写明:
> 1. 现有方法卡在哪?
> 2. 论文认为根因是什么?
> 3. 论文提出什么样的新视角?

## 方法论与核心思路

> 用 3-5 段把方法讲清楚。建议结构:
> 1. 整体框架(架构图的关键模块)
> 2. 关键创新点 1(为什么能解决上面的问题)
> 3. 关键创新点 2
> 4. 训练 / 推理流程
> 5. 与已有方法的本质区别

**架构图(可选)**:描述清楚模块间的数据流

## 核心公式提取

> 提取 2-5 个最关键的数学公式,用 LaTeX 写明含义:

$$
\mathcal{L}_{\text{key}} = \mathbb{E}_{(s,a) \sim \mathcal{D}} \left[ f(s, a; \theta) \right]
$$

**符号解释**:
- $\mathcal{L}$ —— 损失函数
- $s, a$ —— 状态 / 动作
- $\theta$ —— 模型参数

## 关键成果与贡献

> 用 1-3 段陈述实验结果,**要带具体数字**:
> - 相比 SOTA 在 benchmark X 上提升 N%
> - 在 N 个任务上的平均成功率
> - 推理速度 / 训练成本 / 数据需求

## 局限性与未来展望

> 论文自述的局限(不要替论文"谦虚",原文写了就写)
> + 我们阅读时发现的潜在问题(实验设计、对照组、复现难度等)
> + 未来方向(论文提的 + 我们的推断)

## 复现线索

- **代码**: <开源? 哪个框架?>
- **数据**: <用了哪些数据集 / 多少小时>
- **硬件**: <几张 GPU / 训练时长>
- **依赖**: <关键依赖包>
- **复现难度评估**: <低 / 中 / 高,理由>

## 应用与行业映射(本项目强制要求)

### 1. 应用方向

- 工业 / 物流 / 服务 / 家庭 / 医疗 / 特种?
- 哪些场景受益最大?

### 2. 行业玩家

- 谁在做?谁会受益?
- 列出 3-5 家公司 + 它们的对应产品线

### 3. 替代方案

- 现有方案是什么?
- 成本量级 / 性能对比

### 4. 量产壁垒

- 硬件成本、训练数据、可靠性、安全认证
- 卡在哪一环?

### 5. 时间窗估计

- **1 年内(2026)**:能否落地?在哪些场景?
- **3 年内(2028)**:能否规模化?价格预期?
- **5 年内(2030)**:能否成为主流?潜在颠覆性?
